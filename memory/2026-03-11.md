## 关键问题

### 1. SQL 查询问题
- 用户询问 `biz in (:biz)` 当 biz 为空时能否查到数据
- **结论**: 不能，空集合会导致 `in ()` 语法错误或 `in (null)` 永远为假
- 建议参考 `excludeStreamNames` 的处理方式，在 Java 层判断空集合后动态拼接 SQL

### 4. Skill 加载机制
```
~/.openclaw/workspace/skills/*/SKILL.md
  ↓ loadSkillEntries()
  ↓ formatSkillsForPrompt()
  ↓ skillsPrompt (文本)
  ↓ buildEmbeddedSystemPrompt()
  ↓ 注入到 system prompt
  ↓ 发送给大模型
```

大模型根据 skillsPrompt 了解可用技能，决定是否调用某个技能。

### 5. 模型配置
当前会话使用：

- Model: `qwen3.5-plus` (通义千问 3.5 Plus)

模型解析优先级：
1. session entry.model + entry.modelProvider (运行时覆盖)
2. session entry.modelOverride + providerOverride (存储覆盖)
3. agents.defaults.model (agent 默认)
4. config agents.list[].model (配置默认)
5. 全局 default_model (fallback)
